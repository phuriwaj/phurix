The problem with Figma variables is not the feature — it is the workflow gap between creating them and actually using them.

What made variables click for us was treating them as a **design decision journal** first and a tooling feature second. Every variable gets a short description in the variable description field explaining when and why you would use it. That single habit changed how the team discussed design decisions during critique, because you could point to a variable instead of re-litigating the same spacing conversation for the twelfth time.

The two shortcuts that save an hour a week: first, `Option + 2` cycles through colour modes in prototype view, which sounds trivial until you stop manually switching between light and dark preview. Second, right-click any component and "Go to variable" takes you directly to the source — indispensable when debugging why a button looks wrong in one state but not another.
